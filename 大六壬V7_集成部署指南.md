## 🚀 两步部署

### 第1步：替换领域策略文件

**原文件：** `liuren_domain_strategy.js`（仅有4域+通用）
**替换为：** `liuren_domain_strategy_all.js`

```bash
# 备份原文件
cp liuren_domain_strategy.js liuren_domain_strategy.js.bak

# 替换
cp liuren_domain_strategy_all.js liuren_domain_strategy.js
```

### 第2步：在HTML末尾添加一行

打开 `5.18github大六壬复制.txt`，在 `</body>` 之前添加：

```html
<script src="./liuren_integration_v7.js"></script>
```

如果保持原文件名，最终在编辑器中看到的应该是这样的：

```html
    <!-- ⬇️ 新增：V7集成补丁 -->
    <script src="./liuren_integration_v7.js"></script>
</body>
</html>
```

## ⚠️ 常见问题

### Q: 加载报错 "404 Not Found"
确保文件路径和目录结构与上一节一致。`liuren_integration_v7.js` 中默认路径：
- `./uploads/liuren_wangshuai.js`
- `./uploads/liuren_dungan.js`
- `./liuren_bomb_rules.js`
- 等

如果已把所有文件放在同一级，修改 `liuren_integration_v7.js` 中的路径即可。

### Q: 页面未显示V7面板
检查 `window.STATE.currentRun` 是否存在——需先成功排盘一次。V7面板会在排盘完成后自动追加到 `#out_board` 容器中。

### Q: 分类下拉框没有新选项
手动检查控制台是否有错误。如果 `liuren_integration_v7.js` 加载成功但下拉框未更新，可能是选择器ID不同，在控制台执行 `document.getElementById('in_cat')` 确认ID。

### Q: 不想替换原领域策略文件
可以在 HTML 中同时加载两个文件——先加载原版，再加载合并版（合并版会覆盖同名key）：

```html
<script src="./liuren_domain_strategy.js"></script>        <!-- 原版 -->
<script src="./liuren_domain_strategy_all.js"></script>    <!-- 覆盖/扩展 -->
<script src="./liuren_integration_v7.js"></script>
```

```
liuren_integration_v7.js (热补丁入口)
│
├── liuren_wangshuai.js    (旺衰引擎)
├── liuren_dungan.js       (遁干系统)
│
├── liuren_nianming.js     (年命系统 → 独立无依赖)
│
├── liuren_bomb_rules.js   (Bomb Rules → 依赖WUXING常量)
├── liuren_quant_engine.js (E值引擎 → 依赖旺衰+遁干)
│
├── liuren_domain_strategy_all.js (领域策略 → 独立)
└── liuren_ai_prompt_engine.js   (AI提示词 → 依赖量化+Bomb)
```

所有新模块挂载到 `window` 全局对象，以 `WANG_SHUAI_ENGINE`、`DUN_GAN_ENGINE`、`QUANT_ENGINE`、`BOMB_RULES`、`NIAN_MING_ENGINE`、`AI_PROMPT_ENGINE` 命名。
